Wetrocloud RAG-as-a-Service API

The **RAG API** provides endpoints for managing collections, inserting data, and querying resources for Retrieval-Augmented Generation.

Create Collections

This endpoint allows you to generate a unique **collection ID**, which serves as a container for all your resources. This ID is required for all subsequent insert, query, and delete operations.

API Details

  • Method: POST
  • Endpoint: /v1/collection/create/

Required Parameters (in Request Body)

Parameter Type Description
collection_id String The unique ID you want to assign to the new collection.

Successful Response Example

{
  "collection_id": "my_unique_collection_id",
  "success": true
}

List All Collections

Use this endpoint to retrieve a list of all existing collections associated with your account.

API Details

  • Method: GET
  • Endpoint: /v1/collection/all/
  • Parameters: None (Pagination is supported, up to 20 items per request)

Successful Response Example

{
  "count": 5,
  "next": null,
  "previous": null,
  "results": [
    {
      "collection_id": "175af194-2784-4090-bcec-cefe56ddd222",
      "created_at": "2024-12-07T12:23:03.196238Z"
    },
    {
      "collection_id": "4fdb1296-adc8-463c-834d-b76c3831e1f0",
      "created_at": "2024-11-24T14:59:24.047852Z"
    }
  ]
}

Inserting Resources (Files)

This operation uploads a resource (such as a file, URL, or plain text) into a specified collection for indexing and retrieval.

Note: When calling the API directly, all files must be provided as publicly accessible **URLs**. File paths are only supported via the official SDKs.

API Details

  • Method: POST
  • Endpoint: /v1/resource/insert/

Required Parameters (in Request Body)

Parameter Type Description
collection_id String The unique ID of the collection to insert the resource into.
resource String The URL of the file, or the file path if using an SDK.
type String The type of resource being inserted (e.g., "file").

Successful Response Example

{
  "resource_id": "79621681-b680-4918-948d-d7f8c482426a",
  "success": true,
  "tokens": 35
}

Querying Collections (Free Text)

Query your indexed resources within a collection to retrieve a natural language (free text) response based on the RAG process.

API Details

  • Method: POST (Implied for execution)
  • Endpoint: /v1/query/ (Example)

Required Parameters (in Request Body)

Parameter Type Description
collection_id (required) String The unique ID of the collection to query.
request_query (required) String The search term or question to ask the collection.
stream (optional) Boolean Set to True to enable streaming response. Defaults to False.

Successful Free Text Response Example

{
  "response": "The sales for Q1 increased by 20% compared to last year, with online channels contributing significantly.",
  "tokens": 120,
  "success": true
}
